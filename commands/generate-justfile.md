# Generate Justfile Command

Generate a comprehensive justfile for container-based projects with standardized commands leveraging just's first-class parameter support.

## Instructions

### Step 1: Analyze Project

1. **Check for existing justfile**
   - If exists, ask user: "justfile already exists. Overwrite or merge?"

2. **Detect project type**
   - Look for `docker-compose.yml` or `docker-compose.yaml`
   - Look for `Dockerfile` or `.deployment/Dockerfile.*`
   - Identify services from docker-compose
   - Check for deployment scripts (`.deployment/build-and-push.sh`, etc.)

3. **Detect language/framework** (for dev commands)
   - Go: `go.mod`
   - Python: `pyproject.toml`, `requirements.txt`
   - Node/TS: `package.json`

### Step 2: Gather Configuration

Ask user for missing information:

1. **Registry** (if deployment scripts not found):
   ```
   What is your Docker registry? (e.g., ghcr.io/username, docker.io/username)
   ```

2. **Services** (if not detected from docker-compose):
   ```
   What services does this project have? (comma-separated)
   Example: api, worker, web
   ```

3. **Deployment script location** (if not found):
   ```
   Do you have existing deployment scripts?
   - Yes, at: [path]
   - No, generate inline commands
   ```

### Step 3: Generate Justfile

Generate justfile with the following structure:

```just
# ============================================================================
# PROJECT JUSTFILE
# ============================================================================
# Generated by claude-autokit generate-justfile command
#
# Usage: just --list
# ============================================================================

# Settings
set shell := ["bash", "-cu"]
set dotenv-load

# ============================================================================
# VARIABLES
# ============================================================================

registry := env("REGISTRY", "<<REGISTRY>>")
version := env("VERSION", `date +%Y%m%d`)
services := "<<SERVICES>>"

# ============================================================================
# CONTAINER MANAGEMENT
# ============================================================================

# Start all containers
up:
    docker compose up -d

# Stop all containers
down:
    docker compose down

# Restart all containers
restart:
    docker compose restart

# View container logs (follow mode)
logs:
    docker compose logs -f

# View specific service logs
logs service:
    docker compose logs -f {{service}}

# List running containers
ps:
    docker compose ps

# Shell into specific container
shell service:
    docker compose exec {{service}} sh

# Remove stopped containers and dangling images
clean:
    docker compose down --remove-orphans
    docker image prune -f

# Deep clean (volumes, networks, images)
prune:
    docker compose down -v --remove-orphans
    docker system prune -af

# ============================================================================
# DOCKER DEPLOYMENT
# ============================================================================

# Build all Docker images
docker-build:
<<DOCKER_BUILD_CMD>>

# Build specific service
docker-build service:
<<DOCKER_BUILD_SERVICE_CMD>>

# Build and push all images to registry
docker-push:
<<DOCKER_PUSH_CMD>>

# Build and push specific service
docker-push service:
<<DOCKER_PUSH_SERVICE_CMD>>

# Pull latest images from registry
docker-pull:
    docker compose pull

# Full deploy (build, push, notify)
docker-deploy: docker-push
    @echo "Deployment complete!"
    @echo "To update your stack, pull and redeploy in your orchestrator."

# ============================================================================
# DEVELOPMENT COMMANDS
# ============================================================================

# Start development environment with optional arguments
dev *args:
<<DEV_CMD>>

# Run linter
lint:
<<LINT_CMD>>

# Format code
fmt:
<<FMT_CMD>>

# ============================================================================
# HELP & INFO
# ============================================================================

# Show version info
version:
    @echo "Registry: {{registry}}"
    @echo "Version:  {{version}}"
    @echo "Services: {{services}}"
```

### Step 4: Language-Specific Commands

**For Go projects:**
```just
# Start development environment with optional arguments
dev *args:
    go run ./cmd/main.go {{args}}

# Run tests with optional arguments
test *args:
    go test {{args}} ./...

# Run linter
lint:
    golangci-lint run ./...

# Format code
fmt:
    go fmt ./...

# Build binary
build:
    go build -o bin/app ./cmd/main.go
```

**For Python projects:**
```just
# Start development environment with optional arguments
dev *args:
    python -m uvicorn app.main:app --reload {{args}}

# Run tests with optional arguments
test *args:
    pytest {{args}}

# Run linter
lint:
    ruff check .

# Format code
fmt:
    ruff format .

# Install dependencies
install:
    pip install -r requirements.txt
```

**For Node/TypeScript projects:**
```just
# Start development environment with optional arguments
dev *args:
    npm run dev -- {{args}}

# Run tests with optional arguments
test *args:
    npm test -- {{args}}

# Run linter
lint:
    npm run lint

# Format code
fmt:
    npm run format

# Install dependencies
install:
    npm install

# Build for production
build:
    npm run build
```

### Step 5: Deployment Command Templates

**If deployment scripts exist:**
```just
# Build and push all images to registry
docker-push:
    #!/usr/bin/env bash
    set -euo pipefail
    if [[ "{{os_family()}}" == "windows" ]]; then
        cmd /c .deployment/build-and-push.bat all
    else
        bash .deployment/build-and-push.sh all
    fi

# Build and push specific service
docker-push service:
    #!/usr/bin/env bash
    set -euo pipefail
    if [[ "{{os_family()}}" == "windows" ]]; then
        cmd /c .deployment/build-and-push.bat {{service}}
    else
        bash .deployment/build-and-push.sh {{service}}
    fi
```

**If no deployment scripts (inline):**
```just
# Build all Docker images
docker-build:
    #!/usr/bin/env bash
    set -euo pipefail
    for service in {{services}}; do
        echo "Building $service..."
        docker build -t {{registry}}/$service:{{version}} -t {{registry}}/$service:latest -f Dockerfile.$service .
    done

# Build specific service
docker-build service:
    docker build -t {{registry}}/{{service}}:{{version}} -t {{registry}}/{{service}}:latest -f Dockerfile.{{service}} .

# Build and push all images to registry
docker-push: docker-build
    #!/usr/bin/env bash
    set -euo pipefail
    for service in {{services}}; do
        echo "Pushing $service..."
        docker push {{registry}}/$service:{{version}}
        docker push {{registry}}/$service:latest
    done

# Build and push specific service
docker-push service: (docker-build service)
    docker push {{registry}}/{{service}}:{{version}}
    docker push {{registry}}/{{service}}:latest
```

### Step 6: Finalize

1. Write the justfile to project root
2. Show summary of generated commands
3. Suggest: "Run `just --list` to see all available commands"

## Output Format

After generating, display:

```
justfile generated successfully!

Commands available:
- Container: up, down, restart, logs [service], ps, shell [service], clean, prune
- Deployment: docker-build [service], docker-push [service], docker-pull, docker-deploy
- Development: dev [args...], lint, fmt
- Help: version

Run `just --list` for complete command listing with descriptions.
```

## Key Just Features to Leverage

1. **First-class parameters**: Use `recipe param:` instead of pattern rules
   - `logs service:` instead of `logs-%:` / `$*`
   - `shell service:` instead of `shell-%:` / `$*`
   - `docker-build service:` instead of `docker-build-%:` / `$*`

2. **Variadic parameters**: Use `*args` for pass-through arguments
   - `dev *args:` allows `just dev --port 8080`
   - `test *args:` allows `just test -v -run TestFoo`

3. **Default parameter values**: Use `recipe param="default":`
   - `build target="release":` allows `just build` or `just build debug`

4. **Required variadic**: Use `+services` for at least one argument
   - `deploy +services:` requires at least one service

5. **Built-in functions**:
   - `os_family()` for OS detection (returns "windows", "unix")
   - `env("VAR", "default")` for environment variables with defaults
   - Backticks for command substitution: `version := \`date +%Y%m%d\``

6. **Shebang recipes**: For multi-line bash scripts
   ```just
   recipe:
       #!/usr/bin/env bash
       set -euo pipefail
       for item in $list; do
           echo $item
       done
   ```

7. **Recipe dependencies**: Use `recipe: dependency`
   - `docker-push: docker-build` runs build before push
   - `docker-push service: (docker-build service)` for parameterized deps

8. **No help target needed**: `just --list` is built-in and shows recipe comments
