# Generate Makefile Command

Generate a comprehensive Makefile for container-based projects with standardized commands.

## Instructions

### Step 1: Analyze Project

1. **Check for existing Makefile**
   - If exists, ask user: "Makefile already exists. Overwrite or merge?"

2. **Detect project type**
   - Look for `docker-compose.yml` or `docker-compose.yaml`
   - Look for `Dockerfile` or `.deployment/Dockerfile.*`
   - Identify services from docker-compose
   - Check for deployment scripts (`.deployment/build-and-push.sh`, etc.)

3. **Detect language/framework** (for dev commands)
   - Go: `go.mod`
   - Python: `pyproject.toml`, `requirements.txt`
   - Node/TS: `package.json`

### Step 2: Gather Configuration

Ask user for missing information:

1. **Registry** (if deployment scripts not found):
   ```
   What is your Docker registry? (e.g., ghcr.io/username, docker.io/username)
   ```

2. **Services** (if not detected from docker-compose):
   ```
   What services does this project have? (comma-separated)
   Example: api, worker, web
   ```

3. **Deployment script location** (if not found):
   ```
   Do you have existing deployment scripts?
   - Yes, at: [path]
   - No, generate inline commands
   ```

### Step 3: Generate Makefile

Generate Makefile with the following structure:

```makefile
# ============================================================================
# PROJECT MAKEFILE
# ============================================================================
# Generated by claude-autokit generate-makefile command
#
# Usage: make help
# ============================================================================

.PHONY: help up down restart logs ps clean prune dev lint fmt \
        docker-build docker-push docker-pull docker-deploy

# Default target
.DEFAULT_GOAL := help

# ============================================================================
# VARIABLES
# ============================================================================

REGISTRY ?= {{REGISTRY}}
VERSION ?= $(shell date +%Y%m%d)
SERVICES ?= {{SERVICES}}

# Detect OS for script selection
ifeq ($(OS),Windows_NT)
    SCRIPT_EXT := .bat
    SHELL_CMD := cmd /c
else
    SCRIPT_EXT := .sh
    SHELL_CMD := bash
endif

# ============================================================================
# CONTAINER MANAGEMENT
# ============================================================================

## Start all containers
up:
	docker compose up -d

## Stop all containers
down:
	docker compose down

## Restart all containers
restart:
	docker compose restart

## View container logs (follow mode)
logs:
	docker compose logs -f

## View specific service logs (usage: make logs-api)
logs-%:
	docker compose logs -f $*

## List running containers
ps:
	docker compose ps

## Shell into specific container (usage: make shell-api)
shell-%:
	docker compose exec $* sh

## Remove stopped containers and dangling images
clean:
	docker compose down --remove-orphans
	docker image prune -f

## Deep clean (volumes, networks, images)
prune:
	docker compose down -v --remove-orphans
	docker system prune -af

# ============================================================================
# DOCKER DEPLOYMENT
# ============================================================================

## Build all Docker images
docker-build:
{{DOCKER_BUILD_CMD}}

## Build specific service (usage: make docker-build-api)
docker-build-%:
{{DOCKER_BUILD_SERVICE_CMD}}

## Build and push all images to registry
docker-push:
{{DOCKER_PUSH_CMD}}

## Build and push specific service (usage: make docker-push-api)
docker-push-%:
{{DOCKER_PUSH_SERVICE_CMD}}

## Pull latest images from registry
docker-pull:
	docker compose pull

## Full deploy (build, push, notify)
docker-deploy: docker-push
	@echo "Deployment complete!"
	@echo "To update your stack, pull and redeploy in your orchestrator."

# ============================================================================
# DEVELOPMENT COMMANDS
# ============================================================================

## Start development environment
dev:
{{DEV_CMD}}

## Run linter
lint:
{{LINT_CMD}}

## Format code
fmt:
{{FMT_CMD}}

# ============================================================================
# HELP & INFO
# ============================================================================

## Show version info
version:
	@echo "Registry: $(REGISTRY)"
	@echo "Version:  $(VERSION)"
	@echo "Services: $(SERVICES)"

## Show all available commands
help:
	@echo ""
	@echo "Usage: make [target]"
	@echo ""
	@echo "Container Management:"
	@grep -E '^## ' $(MAKEFILE_LIST) | grep -A1 'up:\|down:\|restart:\|logs:\|ps:\|clean:\|prune:' | grep '^##' | sed 's/## /  /'
	@echo "  up                 Start all containers"
	@echo "  down               Stop all containers"
	@echo "  restart            Restart all containers"
	@echo "  logs               View container logs (follow mode)"
	@echo "  logs-[service]     View specific service logs"
	@echo "  ps                 List running containers"
	@echo "  shell-[service]    Shell into specific container"
	@echo "  clean              Remove stopped containers and dangling images"
	@echo "  prune              Deep clean (volumes, networks, images)"
	@echo ""
	@echo "Docker Deployment:"
	@echo "  docker-build            Build all Docker images"
	@echo "  docker-build-[service]  Build specific service image"
	@echo "  docker-push             Build and push all images to registry"
	@echo "  docker-push-[service]   Build and push specific service"
	@echo "  docker-pull             Pull latest images from registry"
	@echo "  docker-deploy           Full deploy (build, push, notify)"
	@echo ""
	@echo "Development:"
	@echo "  dev                Start development environment"
	@echo "  lint               Run linter"
	@echo "  fmt                Format code"
	@echo ""
	@echo "Help & Info:"
	@echo "  help               Show this help message"
	@echo "  version            Show version info"
	@echo ""
```

### Step 4: Language-Specific Commands

**For Go projects:**
```makefile
dev:
	go run ./cmd/main.go

lint:
	golangci-lint run ./...

fmt:
	go fmt ./...
```

**For Python projects:**
```makefile
dev:
	python -m uvicorn app.main:app --reload

lint:
	ruff check .

fmt:
	ruff format .
```

**For Node/TypeScript projects:**
```makefile
dev:
	npm run dev

lint:
	npm run lint

fmt:
	npm run format
```

### Step 5: Deployment Command Templates

**If deployment scripts exist:**
```makefile
docker-push:
	$(SHELL_CMD) .deployment/build-and-push$(SCRIPT_EXT) all

docker-push-%:
	$(SHELL_CMD) .deployment/build-and-push$(SCRIPT_EXT) $*
```

**If no deployment scripts (inline):**
```makefile
docker-build:
	@for service in $(SERVICES); do \
		echo "Building $$service..."; \
		docker build -t $(REGISTRY)/$$service:$(VERSION) -t $(REGISTRY)/$$service:latest -f Dockerfile.$$service .; \
	done

docker-push: docker-build
	@for service in $(SERVICES); do \
		echo "Pushing $$service..."; \
		docker push $(REGISTRY)/$$service:$(VERSION); \
		docker push $(REGISTRY)/$$service:latest; \
	done
```

### Step 6: Finalize

1. Write the Makefile to project root
2. Show summary of generated commands
3. Suggest: "Run `make help` to see all available commands"

## Output Format

After generating, display:

```
Makefile generated successfully!

Commands available:
- Container: up, down, restart, logs, ps, shell-*, clean, prune
- Deployment: docker-build, docker-push, docker-pull, docker-deploy
- Development: dev, lint, fmt
- Help: help, version

Run `make help` for detailed usage.
```
